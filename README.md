# URL Shortener

A production-style URL shortener built on a **Bun + Express + MongoDB + Redis** stack, designed with the real concerns of a high-read redirect service in mind: collision-free code generation, cache-first reads, distributed rate limiting, and graceful failure when the cache layer is unavailable.

## Backend Highlights

### 1. Collision-free short codes via atomic counters

Short codes are **not** random strings. Every URL gets a monotonically increasing `sequenceId` generated by an atomic `$inc` on a MongoDB counter document, which is then **base62-encoded** into the user-facing short code.

```ts
const counter = await Counter.findByIdAndUpdate(
  "urls",
  { $inc: { seq: 1 } },
  { returnDocument: "after", upsert: true },
);
const shortCode = base62.encode(counter.seq);
```

**Why it matters**

- Zero collision probability — no retry loop, no `findOne` before insert.
- Short codes stay compact (`0`, `1`, …, `9`, `a`, …, `z`, `A`, …, `Z`, `10`, `11`…) — `~56.8B` unique URLs in 6 characters.
- `upsert: true` means the counter is self-initializing; no migration step required.

### 2. Cache-first redirects with graceful degradation

The hot path (`GET /:shortCode`) hits **Redis first**, with a 24-hour TTL. On cache miss, it falls back to MongoDB and warms the cache on the way back.

```ts
const cachedURL = await redis.get(shortCode);
if (cachedURL) return res.redirect(cachedURL);

const url = await Url.findOne({ sequenceId: base62.decode(shortCode) });
await redis.set(shortCode, url.originalUrl, "EX", config.CACHED_TTL);
return res.redirect(url.originalUrl);
```

**If Redis is down, the API keeps working.** Every Redis call is wrapped in a try/catch that falls through to the database — cache failures degrade performance, not availability.

### 3. Distributed rate limiting (Redis-backed)

Rate limiting uses `express-rate-limit` with a **Redis store** (`rate-limit-redis`), so limits are enforced correctly across multiple API instances behind a load balancer.

| Endpoint          | Limit        | Reasoning                                                    |
| ----------------- | ------------ | ------------------------------------------------------------ |
| `POST /shorten`   | 10 / minute  | Writes are expensive (counter + insert); abuse creates junk. |
| `GET /:shortCode` | 100 / minute | Reads are cheap and should stay generous for real traffic.   |

Responses follow the IETF `draft-7` standard headers (`RateLimit-Limit`, `RateLimit-Remaining`, `RateLimit-Reset`), so well-behaved clients can back off on their own.

### 4. Defensive validation

Every input crosses a regex gate before any I/O:

- **URL format** — rejected at the router layer with a 400, so invalid traffic never touches the DB.
- **Short code charset** — enforced as `[A-Za-z0-9]+` to prevent wasted base62 decodes and arbitrary-string lookups.
- **Scheme normalization** — inputs missing `http(s)://` are auto-prefixed with `https://` so users can type `github.com` and it Just Works.

### 5. Graceful shutdown

`SIGTERM` / `SIGINT` trigger an orderly shutdown: stop accepting new connections, disconnect Mongo, disconnect Redis, exit. This matters in containerized deploys where the orchestrator sends `SIGTERM` and waits for a clean exit before killing the pod.

```ts
const shutdown = async () => {
  server.close();
  await mongoose.disconnect();
  redis.disconnect();
  process.exit(0);
};
process.on("SIGTERM", shutdown);
process.on("SIGINT", shutdown);
```

### 6. Environment-driven configuration

- **`CORS_ORIGIN`** — comma-separated allowlist; trimmed and passed to the `cors` middleware. Unset → CORS disabled entirely.
- **`REDIS_URL`** — falls back to `redis://localhost:6379` for local dev.
- **`MONGODB_URI`** / **`PORT`** — fail-fast at boot: a missing required env var throws before the server ever binds.
- **`/health`** endpoint — liveness probe for Render / Kubernetes / any PaaS.

## Tech Stack

| Layer            | Choice                                        | Why                                                                 |
| ---------------- | --------------------------------------------- | ------------------------------------------------------------------- |
| Runtime          | **Bun**                                       | Native TypeScript execution, fast cold start, built-in `--watch`.   |
| Web framework    | **Express 5**                                 | Battle-tested, minimal, aligns with the Node.js ecosystem.          |
| Database         | **MongoDB** (Mongoose)                        | Document-native fit for URL records; atomic `$inc` for the counter. |
| Cache            | **Redis** (ioredis)                           | Sub-ms reads on the redirect hot path + shared rate-limit state.    |
| Rate limiting    | **express-rate-limit** + **rate-limit-redis** | Distributed, standard headers, pluggable.                           |
| ID generation    | **base62**                                    | Compact, URL-safe encoding of numeric counters.                     |
| Containerization | **Docker** (oven/bun:1-alpine)                | Deployable to Render, Fly, Railway, ECS.                            |
| Frontend         | **React + Vite + TypeScript + Tailwind**      | (Secondary; primary focus here is the backend.)                     |

## API Reference

### `POST /shorten`

Create a new short code for an original URL.

**Request**

```json
{ "originalUrl": "https://example.com/some/long/path" }
```

**Response** — `201 Created`

```json
{
  "_id": "...",
  "originalUrl": "https://example.com/some/long/path",
  "sequenceId": 42,
  "shortCode": "G",
  "createdAt": "...",
  "updatedAt": "..."
}
```

- `400` — invalid URL format.
- `429` — rate limit exceeded (10 / minute).

### `GET /:shortCode`

Redirects (`302`) to the original URL.

- `400` — short code contains non-base62 characters.
- `404` — no URL mapped to this code.
- `429` — rate limit exceeded (100 / minute).

### `GET /health`

Returns `ok`. Intended for platform liveness probes.

## Running Locally

```bash
# Install all workspace dependencies
bun install

# API  — http://localhost:3000
cd apps/api && bun run dev

# Web  — http://localhost:5173
cd apps/web && bun run dev
```

Populate `apps/api/.env` with at minimum:

```env
MONGODB_URI=mongodb+srv://...
PORT=3000
# CORS_ORIGIN, REDIS_URL — optional on localhost (see .env for details)
```

Redis defaults to `redis://localhost:6379` — any local container (`docker run -p 6379:6379 redis`) will do.
